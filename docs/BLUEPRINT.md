## 1. 🎯 رؤية المشروع

### الهدف الأساسي
بناء تطبيق موبايل **احترافي** و**حديث** يُحوّل Moodle من منصة تقليدية إلى تجربة تعلّم ذكية وجذابة، مع لوحة إدارة متكاملة.

### الفئات المستهدفة

| الفئة | الاحتياج |
|-------|----------|
| **الطلاب** | تعلّم سلس، واجهة حديثة، إشعارات فورية، عمل بدون اتصال |
| **المعلمون** | تقييم الواجبات، إدارة الاختبارات، تتبع تقدم الطلاب |
| **المديرون** | إدارة كاملة للمستخدمين/المقررات/التسجيلات من الموبايل |
| **المؤسسات** | تطبيق مخصص بعلامتهم التجارية، بديل للتطبيق الرسمي |

### المبادئ التوجيهية

```
1. Mobile-First — التصميم أولاً للموبايل
2. Offline-Ready — العمل بدون اتصال كأولوية
3. Performance — أقل من 2 ثانية لتحميل أي شاشة
4. Accessibility — دعم ذوي الاحتياجات الخاصة
5. Bilingual — عربي/إنجليزي من اليوم الأول
6. Security — حماية البيانات والجلسات
7. Scalability — دعم آلاف المستخدمين المتزامنين
```

### 3.3 تدفق معالجة الأخطاء

```
Moodle Error Response
    ↓
ErrorInterceptor (يكشف errorcode في JSON)
    ↓
┌─── "invalidtoken" ──→ AuthException ──→ AuthFailure
│                                              ↓
│                                    إعادة توجيه لـ /login
│
├─── "accessexception" ──→ ServerException(403) ──→ PermissionFailure
│                                                        ↓
│                                              رسالة "لا صلاحيات"
│
├─── Network Error ──→ NetworkException ──→ NetworkFailure
│                                                ↓
│                                    AppErrorWidget + زر إعادة المحاولة
│
└─── خطأ آخر ──→ ServerException ──→ ServerFailure
                                          ↓
                                  رسالة الخطأ للمستخدم
```

## 6. 🔌 التكامل مع Moodle API

### بنية الطلب

```
POST https://{server}/webservice/rest/server.php

Parameters:
  wstoken={token}
  wsfunction={function_name}
  moodlewsrestformat=json
  {additional_params}
```

### التوكن

```
POST https://{server}/login/token.php

Parameters:
  username={user}
  password={pass}
  service=mdf_mobile_service

Response:
  {"token": "abc123def456", "privatetoken": "..."}
```

### خريطة الدوال — Feature ↔ API

| الميزة | API Function | HTTP Method |
|--------|-------------|-------------|
| تسجيل دخول | `/login/token.php` | POST |
| معلومات الموقع | `core_webservice_get_site_info` | POST |
| المقررات المسجلة | `core_course_get_enrolled_courses_by_timeline_classification` | POST |
| بحث المقررات | `core_course_search_courses` | POST |
| محتوى المقرر | `core_course_get_contents` | POST |
| قائمة المستخدمين | `core_user_get_users` | POST |
| إنشاء مستخدم | `core_user_create_users` | POST |
| الاختبارات | `mod_quiz_get_quizzes_by_courses` | POST |
| بدء محاولة | `mod_quiz_start_attempt` | POST |
| الواجبات | `mod_assign_get_assignments` | POST |
| الدرجات | `gradereport_user_get_grade_items` | POST |
| المحادثات | `core_message_get_conversations` | POST |
| التقويم | `core_calendar_get_calendar_events` | POST |

### القيود والحدود

```
- Rate Limiting: لا يوجد افتراضياً في Moodle (يمكن تفعيله بإضافة)
- Max Results: بعض الدوال تعيد 100 نتيجة كحد أقصى
- File Upload: حجم الملف حسب إعدادات PHP
- Token Expiry: حسب الإعداد (0 = بلا انتهاء)
```

### إضافة local_mdf_api المخصصة

بعض العمليات غير متوفرة في API الرسمي، لذا نحتاج إضافة مخصصة:

### ممارسات الأمان

| الممارسة | التنفيذ |
|----------|---------|
| **التوكن** | مخزّن في SecureStorage (AES-256/Keychain) |
| **كلمات المرور** | لا تُخزّن محلياً أبداً |
| **HTTPS** | إلزامي للإنتاج |
| **Certificate Pinning** | مخطط (Dio interceptor) |
| **Token Refresh** | إعادة توجيه لـ Login عند انتهاء التوكن |
| **Data Wipe** | مسح جميع البيانات عند تسجيل الخروج |
| **Obfuscation** | `--obfuscate --split-debug-info` عند البناء |
| **ProGuard** | تفعيل لـ Android release |
| **Input Validation** | فحص المدخلات قبل إرسالها |

### أمان الشبكة

```xml
<!-- android/app/src/main/res/xml/network_security_config.xml -->
<network-security-config>
    <base-config cleartextTrafficPermitted="false">
        <trust-anchors>
            <certificates src="system"/>
        </trust-anchors>
    </base-config>
</network-security-config>
```

### توصيات أمنية إضافية

1. **Root/Jailbreak Detection** — منع التشغيل على أجهزة rooted
2. **Screenshot Prevention** — منع لقطات الشاشة في شاشات حساسة
3. **Session Timeout** — إنهاء الجلسة بعد فترة عدم نشاط
4. **Biometric Auth** — إضافة بصمة/وجه كخيار
5. **App Integrity** — Play Integrity API (Android) / DeviceCheck (iOS)

| استهلاك الذاكرة | < 150 MB | عادي |
| حجم APK | < 30 MB | Split ABI |

### استراتيجيات الأداء

```
1. Lazy Loading
   - الشاشات تُحمّل عند الحاجة فقط
   - الصور تُحمّل عند الظهور (CachedNetworkImage)
   - DI بـ registerLazySingleton

2. Caching
   - صور بـ CachedNetworkImage (تخزين على القرص)
   - بيانات بـ Hive (قراءة سريعة)
   - HTTP caching عبر Dio interceptor

3. Pagination
   - تحميل 20 عنصراً في المرة
   - Infinite scroll مع ScrollController
   - Pull-to-refresh لإعادة التحميل

4. Image Optimization
   - تحميل أحجام مصغرة أولاً
   - WebP format من السيرفر
   - Memory cache + Disk cache

5. Widget Optimization
   - const constructors
   - RepaintBoundary للودجات المعقدة
   - ListView.builder للقوائم الطويلة
   - AutomaticKeepAliveClientMixin للتبويبات
```
